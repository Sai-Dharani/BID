Order Summary

Order Summary is essential for both customer service and business operations, as it helps track the progress of orders, manage inventory, and analyze sales performance.

Channel Summary

Channel Summary Provides insights into customer interactions across various devices, including desktop, tablet, and mobile. It helps identify which channels are most effective for driving traffic and sales.

Ticket Summary

Ticket Summary Provides a breakdown of ticket statuses including opened, closed, pending, resolved, and escalated. Additionally, it tracks the number of scheduled cron jobs and their current status.

Cron Job Summary

Cron Job Summary is a scheduled task that runs automatically at specified intervals on Unix-like operating systems. It's commonly used for automating repetitive tasks like data backups, script executions, or system maintenance. Cron jobs are defined in a configuration file called a crontab, which specifies the schedule and the command to be executed.

User Activity Summary

User Activity is key to analyzing customer engagement and retention. User activity can generally be categorized into active and inactive users.

User Registration / Closed Summary

User Registration / Closedtracking user registrations and account closures is essential for understanding customer lifecycle and platform growth.

API Status Summary

API status codes, also known as return codes, are numerical responses that indicate the outcome of a client’s request to a server. They help developers understand whether an API request was successful, encountered an error, or requires further action.
